# proto-to-zod

Generate [Zod](https://zod.dev) TypeScript validation schemas directly from Protobuf definitions — including field validation rules expressed in `.proto` via [protovalidate](https://github.com/bufbuild/protovalidate).

> **Status:** early research / design. The design proposal lives in [`../proto-to-zod-docs/spec.md`](../proto-to-zod-docs/spec.md), backed by the research notes in `proto-to-zod-docs/`.

## Why

Protobuf is the source of truth for our message/object definitions. Today, validation rules either live only in backend code or are duplicated by hand in TypeScript. This project lets us:

1. Declare validation rules **once**, in the `.proto`, next to the field.
2. Generate Zod schemas whose `z.infer<>` output matches the generated protobuf-es TypeScript types.
3. Validate the same rules at runtime in any TS/JS environment (frontend forms, API edges, etc.).

## Goal

```proto
message User {
  string email = 1 [(buf.validate.field).string.email = true];
  int32  age   = 2 [(buf.validate.field).int32 = { gte: 0, lte: 130 }];
}
```

⬇️ generates

```ts
export const UserSchema = z.object({
  email: z.email(),
  age: z.number().int().gte(0).lte(130),
});
export type User = z.infer<typeof UserSchema>;
```
